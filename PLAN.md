## Remaining Items (To Be Maintained or Refactored Later)

### 1. Build System Artifact Imports
- **Location:** `src/features/games/gamesConfig.default.ts`
- **Count:** 3x `@ts-expect-error`, 3x `eslint-disable import/no-unresolved`
- **Reason:** The code imports `.js` files (e.g., `./rpsConfig.js`) that do not exist in the source tree but are generated by the build script from `.default.ts` files (e.g., `rpsConfig.default.ts`). Changing these imports to `.default.js` satisfies the linter/compiler but breaks runtime because the build script renames the output files.
- **Action:** Retain suppressions until the build system (`scripts/build.js`) is refactored to align source and output filenames.

### 2. Intentional Control Regex
- **Location:** `src/core/utils/sanitization.ts`
- **Count:** 1x `eslint-disable no-control-regex`
- **Reason:** The function `sanitizeString` explicitly removes control characters (`\u0000` etc.) to sanitize input. This is the intended behavior.
- **Action:** Retain suppression as "Intentional".

### 3. Type/Linter Conflict
- **Location:** `src/core/commands/commandManager.ts`
- **Count:** 1x `eslint-disable unicorn/no-useless-undefined`
- **Reason:** The `@minecraft/server` API type definition for command callbacks expects a return type of `CustomCommandResult | undefined`. Returning `void` (implicitly `undefined`) causes a TypeScript error `TS2345`. Explicitly returning `undefined` satisfies TypeScript but triggers the linter rule `unicorn/no-useless-undefined`.
- **Action:** Retain suppression to satisfy both the compiler and the API contract.
